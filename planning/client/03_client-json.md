# Phase 2 — Configure client.json

---

## 2.1 — Get client.json from Raspucat

For all Raspucat-managed clients, `client.json` is **generated by the Raspucat admin panel** from
the client's discovery form submission — not filled in manually.

1. Open the Raspucat admin panel → find the approved quote
2. Click **Generate client.json** — this calls `admin-generate-client-json`
3. Save the output to `execution/frontend/app/client.json`

The exported file will have most values pre-filled. The following fields will be `FILL_IN`
and must be completed by you before running `./deliver.sh`:

| Field | Where to get it |
|-------|----------------|
| `SUPABASE_URL` | Supabase dashboard → Settings → API → Project URL |
| `SUPABASE_ANON_KEY` | Supabase dashboard → Settings → API → anon/public key |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase dashboard → Settings → API → service_role key |
| `STRIPE_PK` | Stripe dashboard → Developers → API keys (if Stripe module) |
| `STRIPE_SECRET_KEY` | Stripe dashboard → Developers → API keys — push to Supabase secrets, not stored in client.json |
| `RASPUCAT_ADMIN_TOKEN` | Your Raspucat admin password |
| `BUNDLE_ID` / `APPLE_TEAM_ID` | Apple Developer account (if native_apps module) |

Webhook secrets (`STRIPE_WEBHOOK_SECRET` etc.) stay empty until the site is live — see [06_stripe.md](06_stripe.md).

> `client.json` is gitignored — it contains secrets. **Never commit it.**

**No Raspucat quote?** Copy from the template instead:
```bash
cd execution/frontend/app
cp client.json.example client.json
```

## 2.3 — Update pubspec.yaml

Open `execution/frontend/app/pubspec.yaml` and update two fields. These are not driven by `client.json` and must be changed manually for every client.

```yaml
name: acme_studio          # snake_case version of CLIENT_SLUG — used as the Dart package ID
description: "Luxury hair salon booking app for Acme Studio."  # Replaces "A new Flutter project."
```

> The `description` field appears in the **Play Store / App Store listing** when publishing a
> mobile build, and some PWA tooling surfaces it. Leaving it as `"A new Flutter project."` is
> an immediate red flag during app store review.
>
> The `name` field must be a valid Dart identifier (lowercase, underscores only). It does not
> have to match `CLIENT_SLUG` exactly but should be close.
